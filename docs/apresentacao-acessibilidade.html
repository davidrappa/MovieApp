Acessibilidade não é só cegueira: tremor, baixa visão, braço ocupado, sol forte, idade avançada.

Como o leitor de tela enxerga seu app


O VoiceOver (iOS) e TalkBack (Android) leem a árvore de acessibilidade — não o que está na tela visualmente.

O leitor anuncia
Role → Label → Hint → State
Exemplo esperado
"Inception, nota 8.8, lançado em 2010. Botão. Toque duas vezes para ver detalhes."
Exemplo sem acessibilidade
"Botão. Botão. Imagem. 8.8. 2010. Inception."
🎬 Demo ao vivo Ativar VoiceOver e navegar pelo app

iOS: Ajustes → Acessibilidade → VoiceOver
Android: Configurações → Acessibilidade → TalkBack

Mostrar a tela de populares antes e depois das correções

Padrão 1 — Elemento Interativo

Todo botão precisa de três propriedades. Sem exceção.

✗ Antes
<TouchableOpacity
  onPress={() => goBack()}
>
  <Icon name="arrow-left" />
  <Text>Voltar</Text>
</TouchableOpacity>

O VoiceOver lê: "Voltar. Imagem." — lê tudo separado, sem contexto

✓ Depois
<TouchableOpacity
  accessibilityRole="button"
  accessibilityLabel="Voltar"
  accessibilityHint="Toque duas vezes para
    voltar para a tela anterior"
  minHeight={44}
>
  <Icon accessible={false} />
  <Text accessible={false} />
</TouchableOpacity>

O VoiceOver lê: "Voltar. Botão. Toque duas vezes para voltar."

⚠ Regra: role + label + hint. Filhos decorativos devem ser marcados como accessible={false} para não duplicar.

Padrão 2 — Estado Dinâmico

Quando o visual muda, o label e o estado de acessibilidade precisam mudar junto.

✗ Antes
<TouchableOpacity
  onPress={toggleFavorite}
  accessibilityRole="button"
  accessibilityLabel="Favorito"
>
  <Icon name={isFav ? "heart-fill" : "heart"} />
</TouchableOpacity>

Label fixo — usuário não sabe se já é favorito ou não

✓ Depois
<TouchableOpacity
  onPress={toggleFavorite}
  accessibilityRole="button"
  accessibilityLabel={
    isFav
      ? "Remover dos favoritos"
      : "Adicionar aos favoritos"
  }
  accessibilityHint={
    isFav
      ? "Toque duas vezes para remover"
      : "Toque duas vezes para adicionar"
  }
  accessibilityState={{ selected: isFav }}
>

Label e estado refletem a realidade

⚠ Regra: accessibilityState comunica o estado atual ao leitor. selected, disabled, checked — use sempre que o elemento tiver estado.

Padrão 3 — Agrupar e Esconder

Se o pai descreve tudo, os filhos precisam sumir da árvore de acessibilidade.

✗ Antes
// Card sem agrupar
<TouchableOpacity>
  <Image source={poster} />
  <Text>Inception</Text>
  <Icon name="star" />
  <Text>8.8</Text>
  <Text>2010</Text>
</TouchableOpacity>

Lê: "Imagem. Inception. Imagem. 8.8. 2010." — 5 focagens para 1 card

✓ Depois
// Pai descreve tudo em uma frase
<TouchableOpacity
  accessibilityRole="button"
  accessibilityLabel="Inception, nota 8.8,
    lançado em 2010"
  accessibilityHint="Abra e veja os detalhes"
>
  <Image accessible={false} />
  <View
    accessibilityElementsHidden={true}
    importantForAccessibility="no"
  >
    <Text>Inception</Text>
    ...
  </View>
</TouchableOpacity>

1 focagem, tudo anunciado de uma vez

O problema real: inconsistência

Mesmo projeto, mesmo time, comportamento diferente em cada tela.

✗ search.tsx / favorites.tsx
<FlatList
  data={list}
  renderItem={...}
/>
// Sem nenhuma propriedade de acessibilidade
// O leitor de tela não sabe que é uma lista
✓ index.tsx
<FlatList
  accessibilityRole="list"
  importantForAccessibility="no"
  data={list}
  renderItem={...}
/>
// Leitor anuncia: "Lista. X itens."

Acessibilidade feita sem convenção de equipe vira acidente.
Você precisa de padrão, não de boa vontade individual.

Tamanho mínimo de toque: 44pt

Guideline da Apple Human Interface e Android Material Design.

Beneficia: pessoas com tremor, mobilidade reduzida, dedos grandes, uso com uma mão.

// ✗ Antes
<TouchableOpacity
  style={{ width: 36, height: 36 }}
>
// ✓ Depois
<TouchableOpacity
  style={{ minHeight: 44 }}
  hitSlop={{
    top: 8, bottom: 8,
    left: 8, right: 8
  }}
>
⚠ hitSlop expande a área de toque sem mudar o visual. Use quando o elemento precisa ser pequeno visualmente mas acessível.

Checklist para o dia a dia


✓ Todo elemento tocável tem accessibilityRole + accessibilityLabel + accessibilityHint

✓ Labels mudam com o estado — nunca fixos em botões dinâmicos

✓ Ícones decorativos têm accessible={false} ou accessibilityLabel=""

✓ Listas têm accessibilityRole="list" e importantForAccessibility="no"

✓ Elementos tocáveis têm no mínimo 44x44pt

✓ Componentes compostos agrupam no pai e escondem os filhos

✓ Teste com VoiceOver/TalkBack ligado antes do PR


Ferramenta útil: Accessibility Inspector (Xcode) para auditar sem precisar de dispositivo físico
